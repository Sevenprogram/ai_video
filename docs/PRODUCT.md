# AI 数字人视频生成平台 - 产品文档

## 1. 产品概述

### 1.1 产品定位

AI 数字人视频生成平台是一款自动化视频内容生产工具，能够将用户的文字描述或简单提示转化为专业级数字人讲解视频。整个流程涵盖：AI 文案生成、TTS 语音合成、智能分镜、自动化录屏、数字人视频合成，最终输出带有画中画效果的完整视频作品。

### 1.2 核心价值

- **全自动化**：用户只需提供主题或描述，系统自动完成从文案到视频的全部流程
- **数字人形象**：支持多种卡通形象替换，实现千人千面的数字人效果
- **画中画模式**：数字人作为小窗口叠加在录屏画面右下角，模拟真人讲解场景
- **高效产出**：支持批量生产，大幅降低视频内容生产成本

## 2. 功能特性

### 2.1 核心功能模块

#### 2.1.1 AI 文案生成

支持多种文案输入方式：

- **主题描述生成**：用户提供主题或产品描述，AI 自动生成口播文案
- **视频分析生成**：上传现有视频，AI 分析内容并生成解说文案
- **分镜脚本生成**：基于分镜脚本自动扩展为完整口播文稿
- **本地文件导入**：支持直接加载本地文案文件

#### 2.1.2 文本转语音 (TTS)

- 支持多种语音风格和音色选择
- 自动生成与视频时长匹配的音频
- 可配置语速、音调等参数

#### 2.1.3 智能分镜

AI 自动将文案拆分为多个镜头单元，每个镜头包含：

- 镜头时长
- 页面 URL（录屏目标页面）
- 页面操作指令（固定展示 / 滚动）
- 数字人动作指令（idle / look_side 等）

#### 2.1.4 自动化录屏

- 生成结构化录屏指令
- 调用 OpenClaw 自动化工具执行录屏
- 支持自定义录屏步骤和页面操作
- 自动保存录屏文件到指定目录

#### 2.1.5 数字人视频合成

**Step 1 - 视频拼接**：将多个数字人动作片段按顺序拼接，裁剪至目标时长

**Step 2 - 音频替换**：将拼接后的视频音频替换为 TTS 生成的语音，支持音频时长对齐

**Step 3 - 头部替换**：使用卡通形象（如猪头）替换视频中的人物头部，支持：

- 人脸检测与跟踪
- 头部缩放与位置调整
- 平滑过渡处理

**Step 4 - 画中画合成**：将数字人视频作为小窗口叠加到录屏画面右下角，形成讲解效果

### 2.2 特色功能

#### 2.2.1 智能动作分配

系统自动识别分镜动作类型，为不同镜头分配合适的数字人动作：

- **idle_hands_open**：默认待机动作，作为常规过渡动作被频繁使用
- **look_side_screen**：左右看/看屏幕动作，用于需要互动感的镜头

idle 动作会被智能重复更多次，使视频整体效果更加自然流畅。

#### 2.2.2 多种视频源支持

- 支持本地数字人视频文件夹
- 支持自定义动作片段库
- 支持多种视频格式（mp4, webm, mov, avi, mkv）

#### 2.2.3 断点续跑

中间步骤生成的视频文件会被缓存，支持跳过已完成的步骤，减少重复渲染时间。

#### 2.2.4 Web 界面

提供可视化 Web 界面，支持：

- 实时查看任务进度
- 视频预览与下载
- 参数配置与调整

### 3.2 核心模块说明

#### 3.2.1 main.py - 主流程入口

负责整体工作流的编排和调度，包括：

- `run_workflow()` - 主工作流入口，协调各模块执行
- `run_recording_full_pipeline()` - 录屏完整流程
- `_run_pipeline()` - 视频合成子流程

#### 3.2.2 video_module/pipeline.py - 视频合成流水线

封装完整的视频合成流程：

- 视频片段拼接与裁剪
- 音频替换与时长对齐
- 头部替换处理
- 画中画叠加

关键函数：`build_video()`

#### 3.2.3 video_module/synthesis.py - 视频合成核心

提供底层视频处理能力：

- `replace_audio()` - 音频替换
- `replace_head()` - 头部替换
- `overlay()` - 画中画叠加

#### 3.2.4 video_module/core/head_replace.py - 头部替换

基于 MediaPipe 人脸检测的头部替换算法：

- 人脸检测与关键点定位
- 头部区域分割与替换
- 平滑过渡处理

#### 3.2.5 video_module/core/ffmpeg_fast.py - FFmpeg 加速

基于 FFmpeg 的快速视频处理：

- `concat_and_trim()` - 快速拼接裁剪
- `replace_audio_fast()` - 快速音频替换
- `overlay_fast()` - 快速画中画叠加

### 3.3 技术栈

- **语言**：Python 3.10+
- **视频处理**：MoviePy, FFmpeg, OpenCV
- **AI 能力**：Google Gemini API
- **语音合成**：TTS 服务
- **自动化**：OpenClaw
- **Web 框架**：Flask
- **人脸检测**：MediaPipe

## 4. 使用方法

### 4.1 环境配置

```bash
# 安装依赖
pip install -r requirements.txt

# 配置环境变量
export GEMINI_API_KEY="your-api-key"
export OPENCLAW_API_KEY="your-openclaw-key"
```

### 4.2 命令行使用

#### 完整流程：主题描述 → 视频输出

```bash
python main.py --topic "比特币价格走势分析" --duration 3
```

参数说明：

- `--topic`：视频主题描述
- `--duration`：视频时长（分钟），默认 3 分钟
- `--output`：输出目录，可选
- `--digital-human`：数字人文件夹名称，可选

#### 仅视频合成（已有素材）

```bash
python main.py --mode pipeline \
    --digital-human "my_digital_human" \
    --audio "output/audio.mp3" \
    --output "output/final.mp4"
```

### 4.3 Web 界面使用

启动 Web 服务：

```bash
python web/app.py
```

访问 `http://localhost:5000` 即可使用可视化界面。

### 4.4 数字人素材准备

将数字人视频片段放入 `video_module/digital_human/` 目录下的子文件夹中：

```
digital_human/
├── my_digital_human/
│   ├── idle_hands_open.mp4    # 待机动作
│   ├── look_side_screen.mp4   # 互动动作
│   └── ...
```

文件名中包含 "idle" 的片段会被识别为默认待机动作，在拼接时会被优先、重复使用。

## 5. 配置参数

### 5.1 视频合成参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `target_duration` | 60.0 | 目标视频时长（秒） |
| `audio_as_canonical` | False | 以音频时长为基准 |
| `head_scale` | 1.8 | 猪头相对人脸缩放倍数 |
| `y_offset_ratio` | 0.25 | 猪头垂直偏移比例 |
| `smooth_window` | 5 | 人脸框平滑窗口帧数 |
| `white_thresh` | 240 | 白色背景抠图阈值 |
| `pip_scale` | 0.18 | 画中画宽度比例 |
| `pip_position` | bottom-right | 画中画位置 |
| `pip_margin` | 20 | 画中画边距（像素） |
| `detect_interval` | 30 | 人脸检测间隔帧数 |
| `ffmpeg_preset` | ultrafast | FFmpeg 编码速度 |

### 5.2 分镜参数

| 参数 | 说明 |
|------|------|
| `action` | 数字人动作指令 |
| `scroll_behavior` | 页面滚动指令 |
| `duration` | 镜头时长 |
| `url` | 录屏目标页面 |

### 6.2 中间文件

视频合成过程中会生成中间文件，位于 `{输出目录}/video_work/`：

- `_step1_concat.mp4` - 拼接后的原始视频
- `_step2_audio.mp4` - 替换音频后视频
- `_step3_pig_head.mp4` - 头部替换后视频

使用 `skip_existing=True` 可跳过已存在的中间步骤。

## 7. 常见问题

### 7.1 视频合成失败

- 检查 FFmpeg 是否正确安装
- 确认所有素材文件路径正确
- 查看日志中的具体错误信息

### 7.2 头部替换效果不佳

- 调整 `head_scale` 参数控制头部大小
- 调整 `y_offset_ratio` 控制头部位置
- 确保数字人视频中人物面部朝前

### 7.3 音频与视频不同步

- 使用 `audio_as_canonical=True` 以音频时长为基准
- 检查 TTS 生成的音频时长是否正确
